## SECTION 1 — WHAT IS A CANONICAL SPINE?

Heading:
Structural Backbone, Not Textual Preference

Content:

A canonical spine is the minimal, stable structural framework of a śāstra.

It defines:

• Hierarchical depth  
• Numerical sequencing  
• Address resolution  
• Unit containment  

It does not define:

• Interpretation  
• Translation  
• Edition preference  
• Philosophical meaning  

The spine is structural authority.
Not textual opinion.

## SECTION 3 — SPINE AS ARCHITECTURAL CONTRACT

Heading:
Declaration, Not Assumption

Content:

A spine exists only when explicitly declared.

Each declaration records:

• Work name  
• Hierarchical depth  
• Numerical basis  
• Declaration authority  
• Status  

This prevents silent mutation.

Spines may be superseded —
never edited.

Infrastructure requires immutability.

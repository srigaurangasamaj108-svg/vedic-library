# Commentary Layer Architecture

## 1. Overview

The Commentary Layer stores **traditional Sanskrit commentaries** and their translations.

In classical Sanskrit scholarship, scriptures are not studied in isolation. They are understood through the explanations of great ācāryas who wrote commentaries explaining the philosophical meaning of the text.

Examples of such commentaries include:

• Śrīdhara Svāmī commentary on the Bhāgavata Purāṇa  
• Madhvācārya commentary on the Bhagavad-gītā  
• Baladeva Vidyābhūṣaṇa commentary on Vedānta texts  
• Viśvanātha Cakravartī Ṭhākura commentaries  

The Commentary Layer allows the Vedic Library to preserve these traditional interpretations digitally.

---

# 2. Why the Commentary Layer Exists

Traditional Sanskrit literature always contains multiple interpretive traditions.

Example traditions:

| Tradition | Example Commentator |
|--------|--------------------|
| Advaita | Śaṅkarācārya |
| Viśiṣṭādvaita | Rāmānuja |
| Dvaita | Madhvācārya |
| Gauḍīya Vedānta | Baladeva Vidyābhūṣaṇa |

Each tradition provides a different philosophical interpretation of the same verse.

If commentaries were stored together with translations or purports, it would be impossible to maintain this diversity.

The Commentary Layer ensures that:

• each ācārya is preserved independently  
• philosophical traditions remain intact  
• translations of commentaries can coexist  

---

# 3. Commentary Layer vs Derivative Layer

The system distinguishes between **Derivative Content** and **Commentary Content**.

Derivative Layer contains:

• modern translations  
• word-by-word meanings  
• purports or explanations  

Commentary Layer contains:

• classical Sanskrit commentaries  
• traditional philosophical interpretations  

This separation preserves the scholarly distinction between **translation** and **bhāṣya**.

---

# 4. Commentary Directory Structure

Commentaries are stored inside the derivatives directory.

Structure:


derivatives/commentary/<author>/<language>/


Example:


derivatives/commentary/sridhara/sa/bg.1.1.sridhara.commentary.json


Full path example:


data/itihasa/mahabharata/bhisma-parva/bhagavad-gita/
derivatives/commentary/sridhara/sa/bg.1.1.sridhara.commentary.json


---

# 5. Language Codes

| Code | Language |
|-----|---------|

# 6. Commentary File Naming Convention

Commentary files follow this pattern:


<verseUID>.<author>.commentary.json


Example:


bg.1.1.sridhara.commentary.json


Translation of commentary:


bg.1.1.sridhara.translation.json


---

# 7. Commentary File Schema

Example Sanskrit commentary:

```json
{
  "uid": "bg.1.1.sridhara.commentary",
  "commentary": {
    "language": "sa",
    "author": "Śrīdhara Svāmī",
    "tradition": "Advaita Vedānta",
    "content": "धर्मक्षेत्रे इति धर्मभूमौ कुरुक्षेत्रे..."
  }
}

Fields:

Field	Meaning
uid	unique commentary identifier
language	language code
author	commentary author
tradition	philosophical tradition
content	commentary text
8. Commentary Translation Schema

Example:

{
  "uid": "bg.1.1.sridhara.translation.en",
  "translation": {
    "language": "en",
    "author": "Śrīdhara Svāmī",
    "commentary_author": "sridhara",
    "content": "Śrīdhara Svāmī explains that Kurukṣetra is called dharma-kṣetra..."
  }
}
9. Commentary Loader

Commentaries are loaded by the Commentary Loader.

File:

frontend/src/features/scripture/commentary.loader.ts

Functions:

loadCommentaries()
loadCommentaryTranslations()

These functions:

read the verse index

determine available commentaries

load commentary JSON files

attach them to the verse composition

10. Commentary Index

To avoid scanning directories repeatedly, commentaries are indexed.

Location:

index/commentary/

Example index file:

bg.1.1.commentaries.index.json

Example:

{
  "verse_uid": "bg.1.1",
  "commentaries": [
    {
      "author": "sridhara",
      "languages": ["sa", "en", "hi"]
    },
    {
      "author": "madhva",
      "languages": ["sa", "en"]
    }
  ]
}

This index is generated by the CLI.

11. Commentary Index Builder

Command:

npm run build:commentary-index

Script:

scripts/cli/build-commentary-index.ts

This scans:

derivatives/commentary/

and produces index files.

12. Commentary Rendering

When a verse is loaded:

loadVerseComposition(uid)

the system attaches commentary layers.

Output example:

COMMENTARIES

— Śrīdhara Svāmī (sa)
— Madhvācārya (sa)

COMMENTARY TRANSLATIONS

— sridhara (en)
— sridhara (hi)
— madhva (en)
13. Importance for Vedic Scholarship

Traditional Vedānta study always relies on commentaries.

Without commentaries:

• philosophical interpretation is incomplete
• doctrinal context is missing

This layer ensures the digital library remains faithful to traditional scholarship.

14. Future Extensions

The commentary layer can expand to include:

• sub-commentaries (ṭīkā)
• cross-commentary comparison
• tradition tagging
• philosophical topic indexing

This will eventually allow comparative Vedānta study across traditions.

15. Summary

The Commentary Layer transforms the Vedic Library from a simple translation reader into a full Vedānta commentary system, preserving the intellectual heritage of classical Sanskrit scholarship.

## 1. Purpose

A Clarification Chat exists for **one reason only**:

> To understand existing decisions ‚Äî not to change them.

This chat is the **only place where confusion is allowed**.

It exists to:
- Prevent silent assumption
- Prevent premature decision-making
- Prevent architectural drift
- Protect canonical integrity

---

## 2. What This Chat IS

A Clarification Chat is:

- Conceptual
- Exploratory
- Explanatory
- Referential

It answers questions like:
- ‚ÄúWhy does this rule exist?‚Äù
- ‚ÄúWhat problem does this document solve?‚Äù
- ‚ÄúHow does this interact with another rule?‚Äù
- ‚ÄúWhere is this decision actually made?‚Äù

This chat produces **clarity**, not output.

---

## 3. What This Chat IS NOT (Absolute Prohibitions)

A Clarification Chat MUST NOT:

- Make decisions
- Modify rules
- Invent conventions
- Override documents
- Approve exceptions
- Assign UIDs
- Segment text
- Emit canonical or editorial data
- Create scripts

If a decision becomes necessary ‚Üí **STOP**.

---

## 4. Authority Relationship

This chat:

- Is subordinate to Core Authority Chat
- Does not bind future work
- Does not produce artifacts
- Cannot lock documents

Its only valid output is **understanding**.

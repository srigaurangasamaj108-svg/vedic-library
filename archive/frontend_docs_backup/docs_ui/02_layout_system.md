# LAYOUT SYSTEM

## Structural Experience of the Vedic Knowledge Interface

---

# 1. INTRODUCTION

The layout is not just visual arrangement.

It is:

> The **spatial representation of knowledge itself**

In this system, layout defines:

* how users think
* how they navigate
* how they understand relationships

---

# 2. THE CORE PROBLEM

Most web applications use:

* Single column (blogs)
* Two column (dashboard + content)

These create problems:

### Problem 1: Loss of Context

User cannot see where they are in the larger structure.

---

### Problem 2: Fragmented Thinking

User reads content but does not see connections.

---

### Problem 3: No Simultaneity

User cannot:

* navigate
* read
* apply

at the same time.

---

# 3. THE SOLUTION: THREE-COLUMN LAYOUT

We define a **simultaneous knowledge experience**.

---

## STRUCTURE

| Column | Role                        |
| ------ | --------------------------- |
| Left   | Structural awareness (Tree) |
| Center | Knowledge consumption       |
| Right  | Knowledge application       |

---

## WHY THIS IS CRITICAL

This creates:

> A **triangular cognitive model**

Where the user always knows:

* Where they are (Left)
* What they are reading (Center)
* Why it matters (Right)

---

# 4. LEFT COLUMN — STRUCTURAL ANCHOR

---

## PURPOSE

The left column anchors the user inside the **Vedic corpus hierarchy**.

---

## USER QUESTION IT ANSWERS

> “Where am I in the knowledge system?”

---

## UX ROLE

* Persistent
* Scrollable
* Hierarchical

---

## DESIGN CHARACTERISTICS

* Narrow but deep
* Text-first
* Indentation-based hierarchy

---

## BEHAVIOR

* Expand / collapse nodes
* Highlight active node
* Allow deep traversal

---

## COGNITIVE IMPACT

Prevents:

* disorientation
* shallow navigation

Creates:

* structural understanding

---

# 5. CENTER COLUMN — KNOWLEDGE CORE

---

## PURPOSE

This is the **heart of the system**.

It displays:

* verse
* translation
* commentary

---

## USER QUESTION IT ANSWERS

> “What is the knowledge here?”

---

## DESIGN CHARACTERISTICS

* Wide
* High readability
* Focused layout

---

## CONTENT STRUCTURE

* Title
* Sanskrit (primary emphasis)
* Tabs (layered knowledge)

---

## WHY CENTER IS WIDER

Because:

* reading requires space
* comprehension requires calm

---

## COGNITIVE IMPACT

Enables:

* deep reading
* contemplation
* absorption

---

# 6. RIGHT COLUMN — APPLICATION LAYER

---

## PURPOSE

This column transforms knowledge into action.

---

## USER QUESTION IT ANSWERS

> “How do I use this knowledge?”

---

## CONTENT TYPES

* Related verses
* Courses
* Guidance
* Seva domains

---

## DESIGN CHARACTERISTICS

* Modular (cards/sections)
* Dynamic (changes per node)
* Slightly lighter visually

---

## COGNITIVE IMPACT

Prevents:

* passive reading

Encourages:

* active engagement
* practical thinking

---

# 7. TOP BAR — GLOBAL CONTROL LAYER

---

## PURPOSE

Provides global controls across the system.

---

## ELEMENTS

* Navigation (Library, Education, etc.)
* Search
* Language selector
* User account

---

## USER QUESTION IT ANSWERS

> “How do I move across the system globally?”

---

## DESIGN CHARACTERISTICS

* Fixed (sticky)
* Minimal height
* Always accessible

---

# 8. FILTER BAR — CONTEXTUAL LAYER

---

## PURPOSE

Personalizes the experience.

---

## POSITION

Below top bar, above content

---

## ELEMENTS

* Age group
* Category
* Seva domain

---

## USER QUESTION IT ANSWERS

> “How does this relate to me?”

---

## BEHAVIOR

* Affects recommendations
* Does NOT change canonical content

---

# 9. RESPONSIVE DESIGN STRATEGY

---

## DESKTOP

* Full 3-column layout
* Independent scroll

---

## TABLET

* Right panel collapsible
* Tree still visible

---

## MOBILE

* Tree → slide-out drawer
* Right panel → bottom sheet
* Content remains primary

---

## IMPORTANT PRINCIPLE

> Never sacrifice content readability

---

# 10. SCROLL BEHAVIOR

---

## INDEPENDENT SCROLL

Each column scrolls independently.

---

## WHY?

Prevents:

* losing position in tree
* losing context

---

## RESULT

User can:

* explore tree
* read content
* scan application

simultaneously

---

# 11. VISUAL HIERARCHY

---

## RULES

1. Center is most prominent
2. Left is structured but subtle
3. Right is supportive

---

## COLOR USAGE

* Center → highest contrast
* Left → muted
* Right → soft cards

---

# 12. TRANSITION BEHAVIOR

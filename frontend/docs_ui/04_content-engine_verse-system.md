# 2. THE CORE PROBLEM

Traditional digital presentations of scripture suffer from:

---

## Problem 1: Information Overload

All elements are shown at once:

* Shloka
* Translation
* Commentary
* Notes

This overwhelms the reader.

---

## Problem 2: Flat Presentation

No distinction between:

* primary knowledge
* interpretation
* application

---

## Problem 3: Lack of Depth Navigation

User cannot:

* explore layers progressively
* focus on one dimension at a time

---

# 3. THE SOLUTION: LAYERED CONTENT SYSTEM

---

## CORE IDEA

Knowledge unfolds in layers.

Therefore, the UI must:

> Reveal knowledge progressively

---

## LAYERS OF A VERSE

Each node contains:

1. Shloka (Original Sanskrit)
2. Transliteration
3. Synonyms (word-by-word)
4. Translation
5. Commentary

---

## WHY THIS ORDER?

Because it reflects:

> Sound → Structure → Meaning → Interpretation

---

# 4. TAB SYSTEM (PRIMARY MECHANISM)

---

## PURPOSE

To separate layers without losing access.

---

## TABS

* Shloka
* Transliteration
* Synonyms
* Translation
* Commentary

## UX BEHAVIOR

* Switching is instant
* Active tab clearly highlighted

## WHY TABS WORK

* reduce cognitive load
* allow focused reading
* preserve depth

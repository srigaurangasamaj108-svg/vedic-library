# VEDIC_LIBRARY_DEVELOPMENT_PHASE_MAP.md
Master Development Evolution of the Vedic Library Platform

Version-Based Development Roadmap

---

# 1. Purpose

This document defines the long-term development evolution of the Vedic Library platform using a version-based roadmap.

The versioning system avoids confusion with existing internal development phases such as:

Track A phases  
Track B infrastructure phases  
UI development phases  

Instead, the platform evolution is expressed through **major platform versions**.

---

# 2. Version Naming Convention

The platform evolves through major versions.

Naming system:

VL-V1 VL-V2 VL-V3 VL-V4 VL-V5
Where:

VL = Vedic Library V = Platform Version
Each version represents a major capability milestone.

---

# 3. VL-V1 — Canonical Reader Platform

This version represents the foundational scripture reading platform.

Core capabilities:

Bhagavad-Gītā reading interface canonical dataset rendering chapter and verse navigation corpus navigation sidebar reading modes basic UI layout
Architecture components:

Next.js application canonical dataset loader corpus registry scripture rendering pipeline
Example route:

/library/bg/2/47
This version establishes the **basic digital scripture reader**.

---

# 4. VL-V2 — Multi-Commentary Scripture Platform

This version introduces commentary traditions.

Capabilities:

multiple commentaries per verse commentary comparison right-side commentary panel commentary selection system
Supported commentators may include:

Śaṅkarācārya Rāmānuja Madhva Śrīla Prabhupāda other sampradāya commentators
UI changes:

right context pane commentary selector side-by-side comparison mode
This transforms the platform into a **scholarly scripture study environment**.

---

# 5. VL-V3 — Knowledge Graph Platform

This version introduces the philosophical concept graph.

Capabilities:

concept registry verse concept tagging concept relationships cross-text discovery concept navigation
Example concept:

karma-yoga
Connected verses:

BG 2.47 BG 3.8 BG 3.19
The platform evolves from **text reading** to **knowledge exploration**.

---

# 6. VL-V4 — Search & Discovery Platform

This version introduces powerful discovery systems.

Capabilities:

full scripture search concept search commentary search cross-text discovery topic-based exploration
Technology layer:

OpenSearch indexing metadata search concept-aware search
Example user queries:

verses discussing detachment references to ātman karma-yoga across texts
This version greatly enhances **research usability**.

---

# 7. VL-V5 — Educational Platform

This version introduces structured learning systems.

Capabilities:

guided reading paths course modules concept learning sequences teacher-led study programs
Example learning path:

karma → karma-yoga → detachment → equanimity
Educational components may include:

lesson systems course structures study groups guided commentary readings
The platform becomes an **educational ecosystem**.

---

# 8. VL-V6 — AI-Assisted Scholarship Platform

This version introduces intelligent research assistance.

Capabilities:

AI concept exploration AI-assisted commentary comparison knowledge graph navigation research assistance tools
Example AI queries:

compare interpretations of BG 2.47 show verses discussing equanimity identify relationships between karma and detachment
AI acts as a **research assistant**, not a doctrinal authority.

---

# 9. VL-V7 — Civilizational Knowledge Platform

The final vision expands beyond scriptures.

The platform represents the knowledge systems of the Vedic civilization.

Domains may include:

Vedānta philosophy Yoga systems Bhakti traditions Dharmaśāstra Āyurveda Jyotiṣa Vāstu Nāṭya Śāstra Arthaśāstra
This transforms the platform into a **civilizational knowledge system**.

---

# 10. Relationship with Track A and Track B

The platform versions sit above the internal architecture tracks.

Track A:

canonical datasets concept registry knowledge graph data
Track B:

platform architecture frontend systems runtime infrastructure
The versions describe **what the platform becomes**, not how it is implemented internally.

---

# 11. Current Development Position

The project is currently between:

VL-V1 and VL-V2
Meaning:

canonical reader completed platform architecture stable commentary system preparation underway
The next major milestone is **VL-V2 (commentary platform)**.

---

# 12. Long-Term Vision

When fully developed, the Vedic Library platform will function as:

a scripture library a philosophical knowledge graph an educational platform a civilizational knowledge system
This would make it one of the most sophisticated digital representations of the Vedic intellectual tradition.

---

# 13. Summary

Platform evolution:

VL-V1 — Canonical Scripture Reader VL-V2 — Multi-Commentary Platform VL-V3 — Knowledge Graph Exploration VL-V4 — Search & Discovery Engine VL-V5 — Educational Platform VL-V6 — AI-Assisted Scholarship VL-V7 — Civilizational Knowledge System
This roadmap defines the long-term direction of the Vedic Library project.
